WhereECH

Settings & privacy

Encrypted Client Hello (ECH) hides the hostname you're visiting from anyone watching the network during the TLS handshake. Without ECH, even though the page contents are encrypted, the site name leaks in the clear. WhereECH tells you which sites support ECH by looking up their DNS records.

WhereECH sends no telemetry, analytics, or browsing data anywhere — only the DNS lookups you configure below.

DNS resolver

To check if a site supports ECH, WhereECH asks a DNS-over-HTTPS provider for that site's HTTPS record. Each lookup reveals one hostname (the site you're visiting) to whichever provider you pick — so choose one you trust with your browsing history.

Enter your NextDNS profile ID (the short code shown in your NextDNS dashboard, e.g. abc123). Lookups will be answered by your own NextDNS configuration, including any filters you've enabled.

Custom resolvers must be HTTPS and speak the JSON DoH protocol (the same format Cloudflare and NextDNS use).

When to check

Confirm with Cloudflare optional

DNS only tells us a site offers ECH — not whether your specific connection actually used it. Cloudflare runs a special endpoint at /cdn-cgi/trace that reports whether the connection it just received from your browser used ECH. Enabling this lets WhereECH verify ECH was really used, but only on Cloudflare-hosted sites.

Cost: one extra request per page to the site you're visiting. Requires permission to read responses from any host (you'll be prompted).

Cache

WhereECH remembers each site's result for 10 minutes (30 seconds for failed lookups) so it doesn't repeat the same DNS query. The cache lives only in memory and is wiped when the browser restarts.
