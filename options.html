Settings & privacy

Encrypted Client Hello (ECH) hides the hostname you're visiting from anyone watching the network during the TLS handshake. Without ECH, even though the page contents are encrypted, the site name leaks in the clear. WhereECH tells you which sites support ECH by looking up their DNS records.

WhereECH sends no telemetry, analytics, or browsing data anywhere — only the DNS lookups you configure below.

Appearance

Choose how WhereECH looks. System follows your OS dark-mode setting.

DNS resolver

To check if a site supports ECH, WhereECH asks a DNS-over-HTTPS provider for that site's HTTPS record. Each lookup reveals one hostname (the site you're visiting) to whichever provider you pick — so choose one you trust with your browsing history.

Enter your NextDNS profile ID (the short code shown in your NextDNS dashboard, e.g. abc123). Lookups will be answered by your own NextDNS configuration, including any filters you've enabled.

Custom resolvers must be HTTPS and support RFC 8484 DNS-over-HTTPS (the standard used by Cloudflare, Quad9, NextDNS, Control D, and most other public DoH providers).

When to check

Per-site lists optional

Restrict or exclude specific hosts from automatic lookups. Manual lookups (clicking the popup or Re-check) always bypass these filters — they only apply to the automatic per-navigation lookup.

Allowlist: if non-empty, only hosts on this list are auto-looked-up. Blocklist: hosts on this list are never auto-looked-up. Both are optional and empty by default. Entries are stored as plain hostnames (not encrypted) — they only contain hosts you typed in on purpose.

      Confirm with Cloudflare optional

      DNS only tells us a site offers ECH — not whether your specific connection actually used it. Cloudflare runs a special endpoint at /cdn-cgi/trace that reports whether the connection it just received from your browser used ECH. Enabling this lets WhereECH verify ECH was really used, but only on Cloudflare-hosted sites.

      Cost: one extra request per page to the site you're visiting.

      Permission required. Enabling this grants WhereECH permission to read responses from every HTTPS website you visit (https://*/*) — the broadest privilege the extension can hold. It's needed because the trace probe has to run on whatever Cloudflare-hosted site you land on, which can't be known in advance. You'll be prompted to approve it, and you can revoke it at any time by turning this off.

      Compare resolvers optional

      When enabled, WhereECH queries every built-in DoH resolver in parallel for each automatic lookup and shows the per-resolver verdict in the popup. Discrepancies (one resolver reports ECH and another doesn't) can indicate DNS manipulation or split-horizon responses.

      Cost: multiplies the number of DoH queries per lookup by the number of built-in resolvers (3, or 4 if you've configured NextDNS). Off by default.

      Status-change notifications optional

      WhereECH can show a system notification when a site you've previously visited starts or stops supporting ECH. The previous status is stored locally per host (just the hostname and its last status — nothing else).

      ECH site history optional

      When enabled, WhereECH keeps a private list of sites it has seen advertising or using Encrypted Client Hello. Nothing about sites without ECH is recorded, and no list is kept at all unless you turn this on.

      How it's protected. The list is encrypted with AES-GCM using a 256-bit key that is generated on your device and stored as a non-extractable CryptoKey in this extension's IndexedDB. The key cannot be read by any JavaScript — it can only be used via the browser's WebCrypto API. Only this extension's origin can see it. The encrypted list itself lives in extension storage; anyone reading the raw profile files sees only opaque ciphertext. Clearing the list also rotates the key so old ciphertext becomes permanently undecryptable. The key also rotates automatically every 30 days.

      Caveat. This protects against casual disk inspection and other extensions, not against someone who can run arbitrary code as you in your browser profile — they could ask the extension to decrypt on their behalf.

        Cache

        WhereECH remembers each site's result for 10 minutes (30 seconds for failed lookups) so it doesn't repeat the same DNS query. The cache lives only in memory and is wiped when the browser restarts. (Disabled entirely if "Disable in-memory cache" is on.)
